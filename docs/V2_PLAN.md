# 司南 Sinan · 第二版(v2)升级设计方案

> 本文是 v2 的**方向与设计草案**,不是承诺排期。所有 v2 能力**必须继续守住 6 条红线**
> (见 `HANDOFF.md` §2 / `sinan-redlines.md`):无未来函数、无虚假回测、不夸大收益、token 永不落明文、
> 无自动真实下单、前端不直连 engine。**任何"更聪明"的能力,都要经诚实引擎验证后才成立。**

v2 的主线一句话:**把"诚实的研究骨架"升级为"会自己发现、会自己解释、会自己提醒"的研究助手**,
而 AI 只负责"提出假设",**最终是否成立由样本外回测说了算**。

---

## 0. 先收尾 v1(进入 v2 前的硬门槛)

这些是 v1 发布前/发布后立即要补的健壮性与诚实化项(评审已确认),不属于 v2 新功能但拦着"正式版":

- 🔴 **api 会话 token 服务端校验**:现在 api 声明了 `x-sinan-token` 却没有 Guard 真正校验
  → 加全局 Nest Guard,`SINAN_IPC_TOKEN` 存在时常量时间比对、不匹配 401(对齐 engine 的 `require_internal`)。
- 🟠 **外壳运行期崩溃自愈**:监控线程 `try_wait` 各 sidecar,死亡→指数退避重启(当前只有启动期探活)。
- 🟡 **设置写接口**:`refresh_interval` 等设置目前前端只读 → 加 `settings.set` 端点,把"自动刷新频率"
  等做成真正可交互(v1 设计还原时因无写接口而诚实保留)。
- 🟡 **train/backtest 转 job 行**:长任务复用 `cache_build` 的 jobs+SSE,可恢复、切导航不丢。
- 🟡 **logs 表无界增长**:加保留窗口/滚动清理。
- 🟢 全局运行指示器、Models/Indicators 实时进度条、建缓存管理 UI(股票池/增量)。

## 1. 算法升级

**目标:在保持"线性可解释基线"的同时,提供更强的非线性模型,且口径仍是诚实样本外。**

- **梯度提升树(LightGBM / XGBoost)** 作为可选 `ModelType`(契约已留 `ModelType` 枚举):
  walk-forward 训练、purge/embargo 不变、样本内外 IC/ICIR 并列;**作为可选 extra 依赖**
  (不进基础包,装机体积不膨胀;无则诚实降级到 ElasticNet)。
- **集成(ensemble)**:线性 + 树模型的等权/按 ICIR 加权融合;融合权重也走样本外评估,不准用样本内调参。
- **稳健回归**:Huber / 分位数回归,降低极端样本影响(配合已有 `winsorize_mad`)。
- ⚠️ 红线:树模型**极易过拟合** → 必须强制展示样本外退化、特征重要性、逐折 OOS IC;
  UI 文案延续"AUC 0.52 是正常"的诚实口径,**绝不把样本内表现当卖点**。

## 2. 因子扩充

**目标:把内置因子从 ~5 个扩到一二十个,且优先"只依赖日线、不卡 Tushare 财务积分"的因子。**

- **价量类(日线即可,不卡积分)**:多周期动量(5/20/60 日)、短期反转、波动率(已实现波动)、
  换手率/流动性(Amihud)、成交额、规模(总市值)、振幅、量比。
- **横截面处理升级**:
  - **中性化(行业/市值)**:打分前对因子做行业、市值残差化(PIT 安全,只用 asof≤T 面板);flag 开关。
  - **去极值/标准化**:已有 winsorize_mad + zscore,补 rank 标准化选项。
- **财务/质量类(需更高积分,诚实降级)**:ROE/毛利率/资产周转等按 `ann_date` PIT;无权限时诚实空。
- **自定义因子并行**:从 DSL 推导回看窗口 → 复用训练那套有界取数 + 多核(当前自定义闭包退串行)。

## 3. 模型优化

**目标:让"模型到底有没有用"一眼可判,并把手动的环节自动化。**

- **模型 vs 等权基线对比(最高优先)**:同口径回测并排,直接显示模型相对等权的超额/IR
  —— 直接回答用户"模型有没有用"。
- **ICIR 自动加权**:因子权重默认按滚动 ICIR 自动分配(当前手动等权/手填),仍 PIT 安全。
- **因子相关性 / 多重共线性报告**:质检页补因子两两相关矩阵,提示冗余因子。
- **因子衰减 / 半衰期(真实)**:由 IC 自相关估计因子信号半衰期 —— 这正是 v1 设计还原时
  因"无真实值"而诚实留空的"半衰期",v2 用 IC 时序**真实算出来**再展示。
- **超参与切分**:purged K-fold、embargo 自适应;超参搜索只在训练窗口内,严禁碰样本外。

## 4. 接入 AI:让 AI 发掘因子(v2 旗舰,且必须守红线)

**核心设计:AI 只"提出假设",诚实引擎"做裁判"。** 这是司南能安全引入 AI 的关键——
AI 生成候选,但**能不能用,由已有的穿越测试 + 样本外 IC 回测决定**,AI 无权宣称收益。

**闭环(全部复用现有诚实管线)**:

1. **AI 提案**:LLM 根据"可用字段 + 仅回看算子(现有 DSL 白名单)"生成一批**候选因子表达式**
   (例如 `rank(mom(close,20) / std(ret,20))`)。可附上"经济学直觉"说明。
2. **沙箱校验**:走现有 `indicators_validate`(AST 白名单 + 仅回看算子,结构上无未来函数)。
3. **诚实回测**:对通过校验的候选,用现有 `factor_quality` 算**真实样本外 IC 均值 / ICIR / 覆盖度 /
   十分位分层**,并跑**穿越测试**(PIT 截断不变式)。
4. **门槛筛选**:只有 OOS IC/ICIR 过阈值、且穿越测试通过的候选才进入"建议因子"列表,
   其余如实丢弃(**不展示样本内"漂亮"结果**)。
5. **人审入库**:用户复核后保存为自定义因子,接入打分/回测(已有 CRUD + 权重)。

**AI 的其它用法**:解释某因子为何有效/失效、批判某模型是否过拟合、给因子组合建议、把自然语言
("找低估值高动量的票")翻译成 DSL。**始终是"AI 提议 + 引擎裁决"**。

**红线与隐私(必须满足,否则不上)**:

- 🔴 **零外联默认**:红线#4 规定"除用户自配数据源 + 签名更新外零外联"。调用云端 LLM = 外联,
  因此 **AI 功能必须 BYO-LLM-key、显式 opt-in、默认关闭**,并在 UI 明确告知"将把因子元信息
  (字段名/算子,**绝不含 token、绝不含你的持仓/账户**)发送到你配置的 LLM"。
- 🔴 **可选本地模型**:优先支持本地 LLM(Ollama 等)路径,做到"AI 也不出本机",最契合司南地基。
- 🔴 **token 仍只走钥匙串**;LLM key 与数据源 token 同等对待(加密存、不落明文/日志/响应)。
- 🔴 **AI 绝不碰下单**(红线#5)、绝不直接写库(红线#6,经 api);AI 产物=候选 DSL 文本,等同用户手写。

## 5. 工程 / 产品 / 数据(v2 配套)

- **更新通道上对象存储**:替掉 v1 的公共代理(gh-proxy)这个脆弱点 —— Aliyun OSS / 腾讯 COS
  托管 `latest.json` + 安装包,签名不变;公共代理仅作兜底。
- **多平台**:macOS / Linux 打包(参数化平台 keyring 子包 + 平台 PyInstaller)。
- **系统托盘 + 关闭确认**;**资讯页(M5)**(spec 已留)。
- **数据增强(需更高 Tushare 积分)**:申万一级行业(更干净 28 类)、北向资金流向(moneyflow,
  补行情页"主力净流入"与板块"资金流入"排序)、财务 PIT 精准化、指数日线(真实基准)、实时报价。

## 优先级建议(v2)

1. **模型 vs 基线对比** + **因子库扩充(价量类)** + **ICIR 自动加权** —— 立刻提升"有没有用"的可判性。
2. **因子中性化** + **因子相关性/半衰期** —— 让质检更专业。
3. **AI 因子挖掘(BYO-key / 本地优先)** —— 旗舰,但务必先把红线/隐私开关做扎实。
4. **LightGBM/ensemble** —— 锦上添花,警惕过拟合。
5. 更新通道对象存储、多平台、数据增强 —— 随积分/资源推进。
